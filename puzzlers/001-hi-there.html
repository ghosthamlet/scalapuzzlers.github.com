--- layout: puzzler title: Hi There! author: Andrew Phillips & Nermin Serifovic sourceName: Daily scala sourceHref: http://daily-scala.blogspot.com/2010/05/return-value-of-block.html scalaVersion: 2.10.0 puzzle: | println( List(1, 2).map { i => i + 1 } ) println( List(1, 2).map { _ + 1 } ) println( List(1, 2).map { i => println("Hi"); i + 1 } ) println( List(1, 2).map { println("Hi"); _ + 1 } ) possibleSolutions: a: code: | List(2, 3) List(2, 3) Hi List(2, 3) Hi List(2, 3) b: code: | List(2, 3) List(2, 3) Hi Hi List(2, 3) Hi Hi List(2, 3) c: code: | List(2, 3) List(2, 3) Hi Hi List(2, 3) Hi List(2, 3) d: code: | List(2, 3) List(2, 3) List(2, 3) List(2, 3) solution: c explanation: | Even though the _ simplification looks the same, in the second case it has a very different effect: the println statement is no longer part of the function body! Instead, the expression is evaluated when determining the function to be passed to map - as with all compound expressions, the last value is returned. ---
